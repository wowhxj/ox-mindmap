#+TITLE: ox-mindmap

Edit a mind map as plain Unicode text, render it to a beautiful image —
all inside a native Org Babel source block.

#+begin_src
,#+begin_src mindmap :file map.svg :width 400
              ╭─ hello
« Root node » ┤
              ╰─ 很好 ── 很坏
,#+end_src

,#+RESULTS:
,#+begin_results
[[file:map.svg]]
,#+end_results
#+end_src

* The idea
[[https://github.com/wowhxj/org-mindmap][org-mindmap]] is wonderful: a mind map is just text you edit in place, and
the engine keeps it pretty. But it lives in a *bespoke* =#+begin_mindmap=
block with its own =C-c C-c= handling and its own (hand-rolled) image and
=#+RESULTS= logic.

ox-mindmap reframes the same map as a *first-class Org Babel source
block*:

#+begin_src
,#+begin_src mindmap :file map.svg
...
,#+end_src
#+end_src

The moment a map becomes a real =src= block, it inherits the entire Org
Babel ecosystem instead of reinventing pieces of it:

- =:file= — write the rendered image, with the link managed for you;
- =:exports code | results | both | none= — control what appears on HTML/LaTeX export;
- =:eval= — gate execution (e.g. =never-export=);
- =:results= / =#+RESULTS= — standard, automatic result block;
- =:var=, =:cache=, =:dir=, =:post=, tangling — all just work;
- attribute injection (=#+CAPTION= / =#+ATTR_*=) — share the *same* setup
  you already use for ditaa / plantuml / dot.

In short: stop maintaining a parallel universe; let Org do what Org does.

* How it works — "model A"
The block body stays the *editable source of truth*: the human-friendly
Unicode map, exactly as in org-mindmap. Pressing =C-c C-c= runs
=org-babel-execute:mindmap=, which does two things:

1. *re-aligns the map in the block body in place* (the equivalent of
   org-mindmap's redraw), and
2. *renders an image* to the =:file= target,

then lets Org Babel insert and maintain the =#+RESULTS= link.

So a single, familiar keystroke gives you both a tidy text map and a
fresh picture — without leaving the =src= block.

* Editing the map interactively
You do *not* have to draw the Unicode connectors by hand. The whole
org-mindmap canvas editor works *inside* the =#+begin_src mindmap= block:
put point on a node and use the keys below — the map redraws itself in
place after every edit.

| Key                       | Action                                                          |
|---------------------------+-----------------------------------------------------------------|
| =TAB=                       | insert a *child* of the node at point (right side at the root)   |
| =C-u TAB=                   | on the *root*: insert a child on the *left* side                 |
| =RET=                       | insert a *sibling* of the node at point (=org-return= off a node) |
| =M-RET=                     | *edit* the text of the node at point                            |
| =M-<left>= / =M-<right>=      | *promote* / *demote* the node; on a root child, *flip* its side  |
| =M-<up>= / =M-<down>=         | swap the node with its *previous* / *next* sibling              |
| =C-c C-c=                   | run the block: re-align the body *and* (re)export the image      |
| =M-x org-mindmap-delete-node= | delete the node at point and all its descendants                |

The root node can branch *both ways*: =TAB= grows the right side, =C-u TAB=
the left. A first-level branch's side is inherited by its whole subtree,
so you only choose left/right once per top-level branch; =M-<left>= /
=M-<right>= on a root child flips that branch (and its descendants) to the
other side.

These come from the bundled engine's =org-mindmap-mode= minor mode, which
ox-mindmap turns on automatically in any buffer that contains a mindmap
block (look for =⅄= in the mode line). Everywhere outside a mindmap block
the keys keep their normal Org meaning — =TAB= cycles, =RET= inserts a
newline, =M-<arrows>= act on Org structure — so the editor stays inert
until point is actually on a map.

=C-c C-c= is deliberately handed to Org Babel (=org-babel-execute:mindmap=)
rather than the engine's own redraw, so re-align, image export and the
=#+RESULTS= link all stay in sync (see "How it works" above).

This behaviour is controlled by =ox-mindmap-interactive= (default =t=); set
it to =nil= to disable the in-block editor and keep only =C-c C-c=.

* Installation — self-contained, no dependencies
ox-mindmap *bundles* the org-mindmap engine (the =org-mindmap*.el= files in
this repository), so it is fully self-contained: install this one package
and nothing else.

#+begin_src elisp
(use-package ox-mindmap
  :vc (:url "https://github.com/wowhxj/ox-mindmap")
  :after ob)
#+end_src

The map engine — parsing the Unicode art, computing the layout, drawing
the SVG, plus the in-place structural-editing commands and
=org-mindmap-mode= — all comes along, since the engine files are part of
this package. ox-mindmap's own code is just the thin Org Babel glue on
top.

#+begin_quote
*Heads-up:* the bundled engine still provides the =org-mindmap= feature.
Do *not* also install the standalone [[https://github.com/krvkir/org-mindmap][org-mindmap]] package next to
ox-mindmap, or the two copies will clash. Use one or the other.
#+end_quote

* Header arguments
| Header        | Meaning                                                       |
|---------------+---------------------------------------------------------------|
| =:file=         | *required* — export target (=.svg= direct; =.png=/=.pdf= via tool) |
| =:width=        | image width, consumed by attribute injection (see below)      |
| =:layout=       | =top= or =centered=                                               |
| =:max-width=    | soft word-wrap column limit, or =auto=                          |
| =:compacted=    | =t= / =nil=                                                       |
| =:paint-depth=  | depth from which to color subtrees                            |
| =:wrap-leaves=  | wrap leaf nodes too                                           |

Defaults (=org-babel-default-header-args:mindmap=):
=:results file link replace :exports results :wrap results=.

For =.png= / =.pdf= output you need one of =rsvg-convert=, =magick=, =convert= or
=inkscape= on =exec-path=; otherwise a sibling =.svg= is written and you are
told about it. =.svg= itself needs no external tools.

* Image attributes (=:width= → =#+ATTR_*=)
ox-mindmap deliberately does *not* emit =#+CAPTION= / =#+ATTR_ORG/LATEX/HTML=
lines itself. Instead it exposes =:width= as a normal header argument and
lands the result inside a =#+begin_results= drawer — so the *same*
attribute-injection setup you already use for other graphics languages
can style mind maps too. Just add ="mindmap"= to the language list:

#+begin_src elisp
(defun insert-attr-decls-at-results (&rest _)
  (when (member (car (org-babel-get-src-block-info))
                '("ditaa" "plantuml" "dot" "mermaid" "d2" "mindmap"))
    ;; read :width, insert #+CAPTION / #+ATTR_* after #+begin_results ...
    ))
(advice-add 'org-babel-execute-src-block :after #'insert-attr-decls-at-results)
#+end_src

(Use a =(&rest _)= signature so it works whether attached as =:after= advice
or as an =org-babel-after-execute-hook=.)

* Customization
- =ox-mindmap-realign= (default =t=): when non-nil, executing a block
  re-aligns its body in place. Set to =nil= to only export the image and
  leave the body untouched.
- =ox-mindmap-interactive= (default =t=): when non-nil, the org-mindmap
  canvas editor (=TAB= / =RET= / =M-RET= / =M-<arrows>=) is active inside
  =#+begin_src mindmap= blocks. Set to =nil= to keep only =C-c C-c=. See
  [[*Editing the map interactively][Editing the map interactively]].
- =org-mindmap-svg-palette= (default a 7-color rainbow set): the branch
  colors for the SVG. Each first-level branch takes the next color in
  order (cycling if there are more branches than colors) and its whole
  subtree inherits it; the root stays neutral
  (=org-mindmap-svg-neutral-color=). Set to =nil= to derive colors from the
  theme via =org-mindmap-color-palette-fn= instead. Which level starts a
  new color is still governed by the =:paint-depth= header (default =0=).
- =org-mindmap-svg-depth-fade= (default =0.16=): each level inside a branch
  keeps the branch hue but is blended this fraction further toward white,
  so a subtree fades from a saturated root to pale leaves instead of being
  one flat color (more visual variety even with few branches). Set to =0=
  for flat per-branch coloring.
- =org-mindmap-svg-box-vgap= (default =6=): pixels of vertical breathing
  room carved out between tightly-stacked node boxes so their borders do
  not touch. Text and connectors stay centered.

* Status
Working. The render engine is the proven org-mindmap one; the Babel layer
and the in-block editing bridge are ox-mindmap's own. =TAB= / =RET= /
=M-<arrows>= now drive the engine's structural editing inside =src= blocks,
with =C-c C-c= reserved for Org Babel's execute/export.
